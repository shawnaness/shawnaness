Wheather

Wheather is a web application/text service that helps you decide how to dress for the weather every morning.

NOTE: We are using a free trial of Twilio to send SMS messages. Because of this, the app currently only works with verified phone numbers. Email me if you'd like to be verified!

Wheather was created during YHack 2017 at Yale University.

As a student at Brown, I constantly find myself frustrated by Providence's weather. Every morning, I check my phone to get an approximate sense of the weather conditions and decide which jacket to wear. However, just seeing the high and low temperature for the day doesn't tell me everything I need to know. What if I don't plan on getting home until 2am? What if it might rain? What if it's super windy or humid? How do these conditions affect how I should dress before I head out for the day?

A small group and I came together at YHack to solve this problem. The name is a portmanteau of "weather" and "whether" - as in whether to wear a jacket or not based on the weather.

We used Python and Flask for the backend and React for the front end. We also used Twilio to send SMS messages to users. The app is hosted on Google Cloud Platform.

An HTML form is used to obtain the user's phone number and zip code, which is sent to the server using jQuery. Our server uses the uszipcode Python package to find the user's latitude and longitude, then uses this information and the Dark Sky weather API to obtain information about temperature and precipitation over the next 24 hours.

We then use this information to construct an informative message for the user, taking into account temperature, precipiation, and UV index. We adjust the apparent temperature so that afternoons - when users are more likely to be outdoors - are weighted more heavily than nighttime temperatures.

This message is then sent to the user as an SMS message via Twilio. The server keeps a record of all subscribed users in a dictionary. Every morning, the server uses an event scheduler to retrieve new information from the API and send a new message to the user.

My main contribution to this project was the front-end development. I built the React Components featured on both our home and submit pages, and I wrote the jQuery ajax request to send the user's information to our server.

I also did the visual design for our page. I decided to use black and white for most of our content, with a accent color of red. I PhotoShopped the background image to be grayscale except for the umbrella, then used the same shade of red for every accent element in the page.

I gained a lot of experience with web development through this project. Though I'm pretty comfortable with React and Python separately, this was my first experience in combining them and sending information between the client and server.

One thing we'd like to add in the future is user personalization - allowing users to input their tolerance to cold and their most active hours, and adjusting our weighting algorithm accordingly. We'd also like to make a mobile version of our site, possibly even expanding into a mobile app sometime in the future.
